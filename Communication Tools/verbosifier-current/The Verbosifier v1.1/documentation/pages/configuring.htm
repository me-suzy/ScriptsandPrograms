Settings in control.php

Number of shouts to display

Set in SHOUT_QTY. The value must be a number not wrapped in quotes (10, not '10'). Default is 10.

Format of date/time

Set in SHOUT_DATE. ../date.txt lists the most commonly used date/time codes. The value must be a string wrapped in quotes. Default is 'Y-m-d'.

Back to contents

Keeping old shouts or deleting as you go

Set in SHOUTS_RETAINED. The value must be a string wrapped in quotes.

A value of 'yes' tells The Verbosifier to keep all old shouts in the database, even though they may not be displayed. This is good if you plan to change the number of shouts on display frequently.

A value of 'no' tells The Verbosifier to delete the oldest shout for every new one that brings the total number of shouts in the database table above the number of shouts to be displayed. For example, if a new post brings the total to 11 and I only want to display 10, the oldest shout will be deleted to bring the total down to 10. This is good if you don't plan to change the number of shouts on display very often.

Naming anonymous users

Set in ANONYMOUS. This is the name given to users who do not put something in the Name field when they shout. The value must be a string wrapped in quotes. Default is 'Anonymous'.

Miscellaneous configuration

Form label language

To change the language the form labels are displayed in, open up verbose.php. The form data is near the top of the page. Just escape any apostrophes/single quotes you add with a backslash (\').

Spamguard

Spamguard uses the power of regular expressions to protect your shoutbox from spamments with a minimum of fuss. To add a new entry to Spamguard, open 'admin/spamguard.php' in a plain-text editor like Notepad. The Verbosifier already comes with a well-compiled spamguard.txt file (98 entries), which should protect you from the vast majority of spamments.

Lines that begin and end with double hashes (## ... ##) are comments, and are ignored in the finished blacklist. It's alright to have spaces within comments. Each blacklist entry appears on its own line. There should be no spaces outside the comments.

A shout will be blocked if a blacklisted string appears within any URL supplied in the shout (e.g. the string 'porn' will block http://www.porn.com, http://porno.net, http://hot-pornos.cz, http://www.pornographic_art.com, http://porn-is-the-devil.org etc. As you can see, Spamguard can block insane numbers of spamments with a single string.

If the spelling of a string changes, it's also possible to block only the part that is common to all spellings. For example, klingeltone is apparently German/Austrian/Europe-Speak (I'm not sure which) for ringtone, but the spelling in some spamments is klingelton or even klingelto. Simply enter the string 'klingelto' to block all three of them.

If you feel a string is too common, then it is best to block the domain name instead. Simply leave out the www. and enter the ___.com for maximum protection (entering www.___.com would not block http://___.com).

If The Verbosifier returns a PHP error when you try to add a new shout, check spamguard.txt for comment lines that aren't ended with double hashes.
